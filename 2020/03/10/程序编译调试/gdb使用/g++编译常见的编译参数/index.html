    1. 研究思路

    由于信息传递会有不可避免的失真，所以研究g++最好的资料就是GNU官网提供的手册。
    《Using the GNU Compiler Collection》

    这份资料详细记录了GNU编译器所有功能选项，总共992页，内容太多。人的精力是有限的，知识的海洋是无限的。

    本文翻译罗列手册（版本为9.2.0）的目录，摘录几个有趣的知识点，介绍几个常用编译参数。

    2. 几个有趣的知识点

    1. GCC支持1998年发布的最早的ISO C++98，
    2. GCC支持现代C++标准，包括2011、2014和2017的修订版（也就是C++11、C++14、C++17）
    3. 使用-std=c++xx选项即可指定标准
    4. 在GCC 9.2.0上默认的标准是gnu++14
    5. 程序编译过程主要有4个步骤：预处理（preprocessing）、编译（compilation）、汇编（assembly）、链接（linking）
    6. GCC支持的可选参数涵盖以上4个步骤
    7. GCC可以只输出上面4个步骤的任一阶段的输出
    8. GCC可以关闭或者打开告警
    9. 位置无关代码（PIC,Position Independent code）

    3. 常用编译参数

    告警相关

    -Werror 所有的告警都输出为错误

    -Wfatal-errors 第一次出错时就立即中断程序编译，不再继续编译和打印更多错误信息

    文件生成相关

    -g 生成将操作系统原生支持的debugging信息。GDB可以用此信息进行调试。

    该选项支持不同级别的调试，Level3包括了程序中的宏定义信息

    -D 用于给程序中添加宏定义， -D macro=string，等价于在头文件中定义：#define macro string

    -o file 将输出存储到file中

    -E 在预处理阶段之后停止，不进行编译，直接输出预处理结果。输出cpp程序的预处理结果，便于看清复杂宏定义含义

    -S 在编译后停止，不进行汇编。输出的是汇编代码

    -c 编译或者汇编代码，但不进行链接

    链接相关

    -l 小写l，手动链接库文件。比如 -lm 用于手动链接数学库libm.a

    -fPIC 用于生成位置无关代码，通常和-shared一起使用，用于生成动态库文件

    -shared 用于生成动态库文件

    其他

    -v 显示版本

    –help 显示帮助信息

    -std 指定使用语言版本
